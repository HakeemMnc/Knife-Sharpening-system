# Code Style

> This file is populated by `/init-project` with stack-specific rules.
> Below are universal defaults. Stack-specific rules are added during project initialization.

## Universal Rules

- Use strict typing wherever the language supports it
- Named exports preferred over default exports
- Use `const` / immutable bindings wherever possible
- No unused variables or imports
- Build must pass with zero errors before committing

## File Naming

- Source files: `kebab-case` (e.g., `user-service.ts`, `auth-helpers.py`)
- Components: `PascalCase` (e.g., `UserProfile.tsx`, `LoginForm.vue`)
- API routes follow framework convention
